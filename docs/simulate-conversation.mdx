## What is Simulation?

Simulation is the process where Arksim runs your pre-built scenarios as live conversations against your agent. Each scenario acts as a simulated user with a defined persona, goal, and prior knowledge who drives a multi-turn interaction with the agent until the user's goal is achieved or the turn limit is reached.

The **output** is a set of conversation transcripts you can inspect directly or pass into Evaluation.

## Inputs

Before running a simulation, you need three things in place:

| Input            | Description                                                                                                                                                         |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Scenarios**    | A `scenario.json` file loaded from `input_dir`. Defines user attributes, goals, knowledge, and scenario metadata. [Learn how to write scenarios →](/build-scenario) |
| **Agent config** | `agent_config.json` in your `agent_setup_dir` defines how Arksim connects to your agent and which protocol to use.                                                  |
| **Config file**  | `config.yaml` controlling simulation parameters such as number of conversations, max turns, workers, and model settings.                                            |

---

## Configuration

```yaml
# Simulation configuration for conversations
num_conversations: 5          # Number of conversations to simulate
max_turns: 5                  # Maximum turns per conversation
num_workers: auto             # Parallel workers 

# Directories
agent_setup_dir: ./examples/bank-insurance
input_dir: ./results/scenario
output_dir: ./results/conversation

# Model configuration
model: gpt-5.1
provider: openai

# Optional settings
simulated_user_behavior_instructions: []   # Custom instructions for user behavior
display_time_delay: false                  # Replay conversations with time delay
```

<Tip>
  Set `num_workers: auto` to automatically parallelize across all conversations, or specify a fixed number to control load on your agent endpoint.
</Tip>

## Agent Compatibility

Arksim supports two agent protocols out of the box:

| Protocol                                                                            | When to use                                                                                   |
| ----------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| [**Chat Completions**](https://developers.openai.com/api/reference/resources/chat/) | Any agent that accepts OpenAI-compatible requests (OpenAI, Azure OpenAI, or a custom wrapper) |
| [**A2A**](https://a2a-protocol.org/latest/)                                         | Agents exposed via the Agent-to-Agent protocol                                                |

Your `agent_config.json` specifies which protocol to use and how to connect.

## Running a Simulation

<Steps>
  <Step title="Set up your config file">
    Create a `config.yaml` file with your simulation parameters:

    ```yaml
    num_conversations: 5
    max_turns: 5
    num_workers: auto
    agent_setup_dir: ./examples/bank-insurance
    input_dir: ./results/scenario
    output_dir: ./results/conversation
    model: gpt-5.1
    provider: openai
    simulated_user_behavior_instructions: []
    display_time_delay: false
    ```
  </Step>
  <Step title="Run the simulation">
    <Tabs>
      <Tab title="CLI">
        Run a simulation with the default configuration:

        ```bash
        arksim simulate config.yaml
        ```

        To specify a scenario directory explicitly:

        ```bash
        arksim simulate config.yaml --input-dir ./results/scenario
        ```
      </Tab>
      <Tab title="Python">
        ```python
        from arksim.simulation_engine import Simulator, SimulationParams
        from arksim.config import AgentConfig
        
        scenarios = Simulator.load_scenarios("examples/path/to/scenario.json")
        agent_config = AgentConfig.load("examples/path/to/agent_config.json")
        
        engine = Simulator(
        agent_config=agent_config,
        scenarios=scenarios,
        params=SimulationParams(num_conversations=30, max_turns=10, num_workers=10),
        )
        
        conversations = engine.simulate()
        ```
      </Tab>
    </Tabs>
  </Step>
  <Step title="Inspect your output">
    Simulation writes a `conversations.json` file to your `output_dir`. This contains the full transcript of every conversation, ready to inspect or pass into [Evaluation](/evaluate-conversation).
  </Step>
</Steps>

## Output

Simulation produces **one primary output file**: `conversations.json`.

This file contains the full transcript of every conversation, including the complete message history, scenario ID, system prompt, user profile, goal, knowledge, user attributes, and all agent and simulated user messages.

### Output Schema

Describes the structure of `conversations.json`, the file produced at the end of a simulation run.

<ResponseField name="schema_version" type="string" required>
  Version of the `conversations.json` schema (e.g., `"v2"`).
</ResponseField>

<ResponseField name="generated_at" type="string" required>
  ISO-8601 UTC timestamp of when the file was generated.
</ResponseField>

<ResponseField name="simulator_version" type="string" required>
  Arksim package version that produced this output.
</ResponseField>

<ResponseField name="conversations" type="list[ConversationItem]" required>
  List of conversation records. Each item represents one full simulated conversation.

  <Expandable title="ConversationItem fields">
    <ResponseField name="conversation_id" type="string" required>
      Unique ID for this conversation.
    </ResponseField>
    <ResponseField name="scenario_id" type="string" required>
      Identifier of the scenario used to generate the conversation.
    </ResponseField>
    <ResponseField name="conversation_history" type="list[ConversationMessage]" required>
      Ordered list of all messages in the conversation.

      - `turn_id` _(int, required)_: Turn index (one user message + one assistant message = one turn).
      - `message_id` _(string, required)_: Unique message ID.
      - `role` _(string, required)_: Either `"sim_user"` or `"assistant"`.
      - `content` _(string, required)_: The message text.
    </ResponseField>
    <ResponseField name="sim_user_prompt" type="SimUserPrompt" required>
      Stores the rendered prompt template and variables used to drive the simulated user. Useful for traceability and debugging.

      - `sim_user_prompt_template` _(string)_: The Jinja template used to construct the simulated user's system prompt.
      - `variables` _(dict[string, Any])_: Key-value map used to render the template. Includes `knowledge`, `user_goal`, `sim_user_profile`, and `agent_context`.
    </ResponseField>
  </Expandable>
</ResponseField>
